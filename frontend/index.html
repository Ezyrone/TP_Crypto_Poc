Attaques

⚠️ alg:none bypass
01 Critique

Un token avec "alg":"none" ne contient pas de signature. Un serveur vulnérable l'accepte sans vérification, permettant de forger n'importe quel rôle.

🔀 Confusion RS256 / HS256
02 Critique

La clé publique RSA est exposée. L'attaquant l'utilise comme secret HMAC pour signer un token HS256. Le serveur vulnérable l'accepte faute de forcer l'algorithme attendu.

🔨 Brute-force HS256
03 Haute

Avec un token JWT intercepté, un attaquant peut tester des secrets hors-ligne à vitesse élevée. Le secret "secret" est découvert via dictionnaire.
